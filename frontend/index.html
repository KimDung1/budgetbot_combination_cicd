🛡️ 2-Layer Protection Architecture
1

Layer 1 — Input Validation

Runs first, before any AI processing.

Enforces strict rules on raw input characters and length to block structurally malformed data before it ever reaches pattern matching or the AI model.

Max 255 chars Blocks < > { } $ Blocks % ^ | ` ~ Prevents XSS
2

Layer 2 — Threat Pattern Matching

Runs second, checks semantic patterns.

Scans text against a library of known attack patterns. Any match immediately blocks the input — AI model is never invoked.

SQL Injection Command Injection Template Injection {{ }} Prompt Injection
✅ Result: Malicious inputs are stored in DB with confidence=blocked for audit. They are never sent to AWS Bedrock, protecting both security and cost.
